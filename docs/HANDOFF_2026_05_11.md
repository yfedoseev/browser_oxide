## What landed (in commit order)

### `4c6d364` — feat(timer,page,net): +4 sites → 116/126

Recovered all 4 R5 regressions with two engine fixes:

1. **`crates/js_runtime/src/js/timer_bootstrap.js`** —
   `UNREF_THRESHOLD_MS` raised 1000 → 2000 ms. R5's `5c649e1` unref'd
   long setTimeouts to free twitter/x.com from the
   `op_timer_sleep`-perpetual-pending pin, but cost macys/ria/threads
   whose hydration uses ~1.5 s render-critical timers. 2000 ms is
   the Pareto-optimal single threshold: keeps twitter+x (hundreds of
   sub-second timers + occasional 5–60 s analytics retries) AND
   preserves 1–2 s SPA hydration callbacks. Tried 3000 ms → ria
   regressed back to THIN, so 2000 ms wins.

2. **`crates/browser/src/page.rs`** — `Page::resolve_url()` returns
   `Some` only for `http`/`https`; `about:blank`, `data:`, `blob:`,
   `javascript:`, `chrome-extension:` etc. return `None`. Every
   caller (scripts, iframes, stylesheets, fetches) skips uniformly.
   The pending-navigation harvester in `Page::navigate` treats
   "unfetchable scheme" as no-op (`return Ok(page)`) instead of a
   hard error. Caught on iphey.com: programmatic iframe src ended
   up as `about:blank` after `Url::join`; downstream `host_str()`
   returned None and we bubbled "no host in URL".

3. **`crates/net/src/lib.rs`** — all five "no host in URL" sites
   now embed the failing URL via `format!("...: {url}")` for
   diagnostics. No behavior change.

### Capture infrastructure that exists today

### What's been reverse-engineered

1. **Kasada's `/tl/error` report wrapper = XOR with key `omgtopkek`**
   (constant across deployments; cracked 05-10)
2. **Kasada's VM = 50 opcode handlers** installed via
   `new Function(body)()` calls (162 per session bootstrap). The full
   opcode table is in the W4a doc with semantic labels (`LOAD-INDEXED`,
   `STORE-INDEXED`, `ADD`, `SUB`, `DIV`, scope-lookup variants,
   `DEFINE-FN`, `TYPEOF`, `NEW-ARRAY`, `CALL-N`, etc.)
3. **XXTEA cipher** used for constant-table decryption (handler #5,
   1170-byte body containing `var d=2654435769`)
4. **The failing opcode is #12 `STORE-INDEXED`** — matches the
   TypeError stack frame `eval line=3 col=66 isEval=true`
5. **The probe pattern**: `e(n)[e(n)] = e(n)`. The middle `e(n)`
   returns the obfuscated key `unjzomuybtbyyhwwkdpkxomylnab` from
   the const table. The first `e(n)` returns the receiver. For our
   engine the receiver is `undefined` 2× per run; for the success
   cases (100+ per run) the receiver is a Kasada-internal Function
   object (verified via `Object.prototype.unjzomuybtbyyhwwkdpkxomylnab`
   getter probe)
6. **117-field error report** decrypted; 7 fields consistently flag
   us as a bot:

| Field | Error result | Cluster context |
|---|---|---|
| `ao` | `TypeError: Invalid attempt to spread non-iterable instance` | `spc, kbk, ao, ddt2, cpl` |
| `cbf` | `TypeError: Cannot read properties of undefined (reading 'toString')` | `uh, ss, cbf, eck, npe` |
| `crs` | `"no error thrown"` (Kasada expected one) | `nppm, npf, crs, npc, ecp` |
| `csc` | `TypeError: Cannot read properties of undefined (reading 'unjzomuybtbyyhwwkdpkxomylnab')` | `snp, dpv, csc, spd, gp` |
| `kl`  | unjz error | `jdo, tn, kl, nps, uad` |
| `nhp` | `"unexpected"` (hash/pattern mismatch) | `nps, uad, nhp, eem, eem2` |
| `bot1225` | unjz error (final verdict) | top-level |

Plus one intermittent: `mrs` (Worker MediaSource — fixed in
`5c2328c` for hygiene; didn't measurably drop in this run, may
need a different fix path)

The mapping requires one of:
1. **Playwright-Chrome reference run** (recommended): load
