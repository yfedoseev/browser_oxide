### §5.3 — FIX-D2 (WebGL 1 vs WebGL 2 context separation)

**Why second:** AWS WAF challenge.js emits BOTH `getContext("webgl")` AND `getContext("webgl2")` and cross-checks. Today our `canvas_bootstrap.js:967` returns the same `WebGLRenderingContext` instance for both; a WebGL 1 request sees WebGL 2 strings + extensions back.

**Data available:** every Camoufox row in `webgl_data.db` has separate `webGl:*` (WebGL 1) and `webGl2:*` (WebGL 2) sub-objects. The Apple M1 row covers macOS WebGL 1 close to M3 (renderer-substitute the unmasked_renderer string).

**Scope (per `vNext/07_FIX-D2-D3-WebGL.md`):**
1. Extend `GpuProfile` schema to carry BOTH surfaces:
   ```rust
   pub struct GpuProfile {
       // existing fields = the "preferred" / WebGL 2 surface
       ...
       /// WebGL 1.0 surface (separate from the WebGL 2 fields).
       /// None = use same as WebGL 2 (old behaviour, kept for migration).
       pub webgl1: Option<WebGL1Surface>,
   }
   pub struct WebGL1Surface {
       pub version: String,
       pub shading_language_version: String,
       pub extensions: Vec<String>,
       pub params: Vec<(u32, serde_json::Value)>,
       pub shader_precision: Vec<(u32, u32, [i32; 3])>,
   }
   ```
2. Update `canvas_bootstrap.js:925` so `getContext("webgl")` returns an instance of a new `WebGLRenderingContext` (WebGL 1 surface) while `getContext("webgl2")` returns the existing `WebGL2RenderingContext` (WebGL 2 surface). Both classes share the same backing canvas id; the difference is the JS-side parameter/extension lookup map.
3. Update the Rust side to plumb `profile.gpu_profile.webgl1` through the `op_canvas_*` ops as a separate dictionary.

**Files to touch:**
- `crates/stealth/src/gpu.rs` — new `WebGL1Surface` struct + field
- `crates/stealth/src/profile.rs` — default = derive WebGL 1 from same renderer
- `crates/js_runtime/src/js/canvas_bootstrap.js:925-964` — `getContext` dispatch
- `crates/js_runtime/src/js/canvas_bootstrap.js:1050` — currently aliases `WebGL2RenderingContext = WebGLRenderingContext`; replace with a real subclass
- Snapshot tests + chrome_compat updates

**Validation:**

**Expected lift:** +0-2 sites; mostly correctness. AWS WAF cross-checks tighten without crashing.

- **Don't run the full gate from the same IP if Sprint 1 isolation didn't ship cleanly.** Verify `benchmarks/run_sweep_isolated.py` produces consistent results before kicking off the 10 h sweep — single-site smoke first.
