## Major engineering wins

### TLS fingerprint = byte-perfect Chrome 147 (verified)
Captured our actual JA4 via `tls.peet.ws/api/all` and compared against
real headed Chrome 147 from the same residential IP:

```
ours    : t13d1516h2_8daaf6152771_d8a2da3f94cd
chrome  : t13d1516h2_8daaf6152771_d8a2da3f94cd
```

`ja4`, `ja4_r`, `peetprint_hash`, ALPS payload, H2 SETTINGS frame —
all match. **The engine is now indistinguishable from real Chrome at
the TLS layer.** Two fixes shipped:
1. Drop Zlib from cert-compression list (Chrome 147 sends only Brotli)
2. Plain Fisher-Yates shuffle over all 16 extensions (was 3-bucket folkore)

**Implication**: the canadagoose H2→HTTP/1.1 ALPN downgrade is NOT a TLS
fingerprint issue. Verified by side-test: same engine, same IP, gets h2
on google.com / cloudflare.com / tls.peet.ws / canadagoose.com/favicon.ico
(returns 200 + 11.7 KB real favicon). canadagoose downgrades EVERY path
to h1.1 — per-origin Akamai config, possibly IP-tied rate limit.

### Kasada error report wrapper cracked
Algorithm: `base64(json({"data": base64(xor(plain, b"omgtopkek"))}))`.
9-byte repeating XOR, key is a deployment-wide constant (verified across
3 captures). Decryptor: `docs/kasada_ips_analysis/scratch/decrypt_report.py`.

This collapses all future Kasada debugging to a 5-line Python helper.
Used during this session to identify:
- The **CSS calc() math-function fingerprint probe** (1283-byte raw text)
  → fixed via `crates/css_values/src/calc.rs`
- The **`Function.prototype.toString` source leak** (`sfc.funcs` field
  showed our deno_core op names verbatim) → fixed via mask sweep
- The **`MediaRecorder.isTypeSupported` undefined receiver** (`mrs` field)
  → fixed via stub
- The **`_loadGpuProfile is not a function`** internal name leak (`esd.wgl`)
  → fixed via static method + rename to `_g`
- 13 remaining error-bearing fields documented in
  `docs/CANADA_GOOSE_DIAGNOSIS_2026_05_10_PART2.md`

### CSS Values 4 math functions in calc()
Full implementation in `crates/css_values/src/calc.rs` of the math
function set Chrome ships:
- arithmetic: `+`, `-`, `*`, `/`
- comparison: `min/max/clamp`
- stepped: `round` (4 strategies) / `mod` / `rem`
- trigonometric: `sin/cos/tan/asin/acos/atan/atan2`
- exponential: `pow/sqrt/hypot/log/exp`
- sign: `abs/sign`
- constants: `pi/e/infinity/-infinity/NaN`

Wired into `op_dom_get_computed_style` so `getComputedStyle` returns the
*resolved pixel value* (not the unresolved expression). Empirically
verified to drop the CSS calc precision probe from Kasada's error report
(blob count 9 → 8).

### Function.prototype.toString mask sweep
Comprehensive sweep at the end of `dom_bootstrap.js`. Walks every named
own property of ~50 Web API class prototypes (Element, HTMLElement,
Document, Node, all HTMLXXXElement subclasses, AudioContext family,
Worker family, Notification/Bluetooth/StorageManager, MediaRecorder,
MediaSource, MediaSession etc.) plus 25 top-level globalThis functions
(queueMicrotask, fetch, setTimeout, ...). Each function-typed value/
getter/setter gets `_maskFunction` so `.toString()` returns
`function NAME() { [native code] }`. Killed the visible
`op_dom_attach_shadow` source leak.
