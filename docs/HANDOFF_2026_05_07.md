# Session Handoff — 2026-05-07

## Summary of Work
This session pivoted from reverse-engineering Kasada encryption to fixing structural browser gaps that were causing detection on Tier 0/1 sites. Key focus areas were foundational JS engine parity (SharedArrayBuffer) and the Performance Timing API.

### 1. Foundational JS Parity (SAB/Atomics)
- **Fix**: Corrected the gating logic in `cleanup_bootstrap.js`. Previously, `SharedArrayBuffer` was being unconditionally deleted or incorrectly gated, causing it to be missing even in cross-origin isolated contexts.
- **Parity**: The constructor is now correctly hidden on non-isolated pages and present when headers allow, matching Chrome's security model.
- **Verification**: `sab_constructible_with_byte_length` and `sab_instance_is_shared_array_buffer` tests now pass.

### 2. Performance API Hardening
- **Dynamic Resource Timing**: Rewrote `window_bootstrap.js` to dynamically generate `PerformanceResourceTiming` entries from the DOM (scripts, links, images) and active network requests (fetch/XHR).
- **Timing Parity**: Implemented dynamic zeroing of future-dated properties (like `domComplete`) if the document state is not yet `complete`.
- **Instrumentation**: Added hooks to `fetch_bootstrap.js` and the XHR shim to push real timing data into the performance entry list.
- **Vendor Fallbacks**: Ensured critical scripts (Qrator, WBAAS) are always represented in the timing list to satisfy strict vendor checks.

### 3. Solver & Engine Reliability
- **Timeouts**: Increased `op_net_fetch_sync` timeout to **30s** and total navigation budget to **60s** (via `BOXIDE_NAV_BUDGET_MS`) to allow slow solvers to finish.
- **Diagnostics**: Added header logging for challenge responses and reloads in `page.rs` to identify missing signals (e.g., Client Hints).
- **Script Capture**: Added logic to `fetch_ext.rs` to save large sync-fetched JS blobs to `/tmp/fetched_script_<len>.js` for local analysis.
