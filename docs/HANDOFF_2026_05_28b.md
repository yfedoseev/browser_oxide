## Â§4 â€” AWS/duolingo root cause: four layers peeled

The deterministic imdb/amazon-in stub-stick was chased to ground:

1. **NOT a fingerprint bail.** Via the oracle (`aws_capture` â†’ instrument â†’ `awswaf_probe`): fed the live stub, challenge.js ran to completion and **called the token function** (`forceRefreshToken`) with BO's fingerprint. It reads `navigator.plugins` Ă—24, `webdriver`, WebGL unmasked vendor/renderer + `getSupportedExtensions`, `chrome.csi/loadTimes`, `performance.now` â€” and proceeds. **So FIX-D2 / any WebGL or fingerprint fix could never flip AWS** (confirmed by the FIX-D2 measure: imdb/amazon-in unchanged).
2. **challenge.js IS fetched live** (`[seccpt-trace] â€¦ challenge.js status=200 bytes=1370004`) on every nav iteration.
3. **NOT WebAssembly.** `grep challenge.js`: `new Worker` Ă—2 + `new Blob` Ă—2, **zero `WebAssembly.*`**. The PoW runs in a **blob-URL Web Worker**. Probed BO's worker realm and found **`crypto.subtle` was `undefined` in workers** (SecureContext-gated; the worker realm never inherited secure-context). **Fixed (commit `5216336`).** Verified the worker now has `crypto.subtle` (SHA-256 digest works). Basic blob-URL worker round-trips already worked.
4. **The remaining blocker (the real lever):** even after the worker fix, **imdb/duolingo don't flip**, because in the **live navigate path challenge.js produces ZERO async progress** â€” fetched Ă—3, but **no worker spawn, no token POST, no fetch** (verbose `worker_ext`/`fetch_ext` trace). Yet the **offline oracle** (`from_html_with_url` + `run_until_idle(5s)`) runs it fully to `forceRefreshToken`. The difference is the **execution/drain model**: `build_page_with_scripts_init_and_storage` runs external scripts with only a **50 ms inter-script `run_until_idle`** (page.rs:~3535) and then the nav loop re-fetches the stub, before challenge.js's `checkForceRefresh().then(...)` promise chain (and its deferred worker creation) advances.

**Net:** AWS is an engine **self-solve-execution** gap, not stealth. The worker `crypto.subtle` prerequisite is in place; the live-nav async-drain is the last lever.
