## Why 115 is hard from where we are

Three blockers compound:

### 1. AWS WAF per-IP per-region clustering (the dominant lever)

Single-IP measurement: when BO sweeps 8 amazon TLDs back-to-back from the same datacenter IP, AWS WAF deploys per-region (us-west-2, eu-west-1, ap-northeast-1, etc.). The first 3 we hit get tokens (cold WAF state); subsequent regions see "this IP just got 3 tokens, suspicious" and reject. **Verified by data**: the 3 passing (UK, AU, FR = EU + AU AWS regions) vs the 6 failing (CA, com, DE, IN, JP = NA + DE-specific + APAC). Same trial, same fixes, same fingerprint — different outcomes purely by request sequence + per-region state.

### 2. DataDome WASM solver isn't in the public engine

Per CLAUDE.md, the WASM daily-key computation lives in `vendor_solvers` (private companion crate). FIX-DD landed the 3 public-engine primitives (CSP relax, iframe materialize, cookie-watch break-on-solve) so the bundle's self-solve can RUN — but the solver itself, which produces the `datadome=` token, is `vendor_solvers` work. Without it, etsy + yelp + leboncoin-on-bad-days stay at DataDome-CHL.

### 3. Per-non-macOS GPU profiles still have FIX-D-era bugs

FIX-D fixed apple_m3_macos to the captured Chrome 147 M3 fixture. `chrome_148_windows` (nvidia) and `chrome_148_linux` (intel) still use the old `common_params_desktop()` baseline with WRONG `MAX_VIEWPORT_DIMS=[32767,32767]` and mixed WebGL 1+2 extension lists. The routed-best-of-4 median TAKES THE MAX of 4 profiles per site — if windows + linux + iphone profiles all fail amazon-de/in/jp, the routed median can't beat macOS's single number. **FIX-D3** (per-non-macOS GPU validation per [vNext/07](vNext/07_FIX-D2-D3-WebGL.md)) is the unlock.

## What didn't work / things to avoid

- **Single-IP sweep is the worst possible measurement environment for AWS WAF cluster** (verified this session). Don't conclude "fix didn't work" from a single-IP single-trial. The fix IS working; the measurement is hiding it.
- **Wide profile sampler v1 (5 chip configs varying cores independently)** regressed an 8-site AWS WAF sweep from 1/8 → 0/8 because of cross-API inconsistency (12-core CPU + "Apple M3" GPU = M3 is 8-core base). Tightened to M3-family-only per-chip-coherent in FIX-E2.
- **FIX-D applied only to apple_m3_macos** — other GPU profiles still have the FIX-D-era bugs. The routed best-of-4 median CAN'T benefit from a per-site-best-attempt if 3 of the 4 profiles are broken on the same site.
